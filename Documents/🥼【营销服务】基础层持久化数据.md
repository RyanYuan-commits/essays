## 要点记录
### 关于 Guava 的 Cache
Guava 的 Cache 是一个 **内存缓存实现**，提供了线程安全的缓存机制，支持多种缓存策略。
Guava Cache 适用于需要在内存中缓存数据以提高访问速度的场景，特别是当数据的生成成本较高或访问频率较高时。它不适合用作持久化存储，因为缓存中的数据会在程序重启或内存不足时丢失。
>[!info] com.ryan.config.GuavaConfig.java
```java
/**  
 * @author Ryan Yuan 
 * 2024-10-21 11:19 
 * Guava 缓存配置  
 **/  
@Configuration  
public class GuavaConfig {  
    @Bean  
    public Cache<String, String> cache() {  
        return CacheBuilder.newBuilder()  
                .expireAfterWrite(10, TimeUnit.SECONDS)  
                .maximumSize(1000)  
                .build();  
    }  
}
```
### 关于线程池的配置
可以通过这样的方式去配置一个公用的线程池 Bean。
`@EnableConfigurationProperties` 注解用于启用配置属性类的支持，使得这些类可以自动从外部配置源（如 `application.properties` 或 `application.yaml` 文件）中读取属性值。
需要配合一个配置类来使用。（`@ConfigurationProperties` 标记的类）。
>[!info] com.ryan.config.ThreadPoolConfig
```java
@Slf4j  
@EnableAsync  
@Configuration  
@EnableConfigurationProperties(ThreadPoolConfigProperties.class)  
public class ThreadPoolConfig {  
    @Bean  
    @ConditionalOnMissingBean(ThreadPoolExecutor.class)  
    public ThreadPoolExecutor threadPoolExecutor(ThreadPoolConfigProperties properties) {  
        // 拒绝策略，默认为 AbortPolicy        RejectedExecutionHandler handler = switch (properties.getPolicy()) {  
            case "DiscardPolicy" -> new ThreadPoolExecutor.DiscardPolicy();  
            case "DiscardOldestPolicy" -> new ThreadPoolExecutor.DiscardOldestPolicy();  
            case "CallerRunsPolicy" -> new ThreadPoolExecutor.CallerRunsPolicy();  
            default -> new ThreadPoolExecutor.AbortPolicy();  
        };  
        // 创建线程池  
        return new ThreadPoolExecutor(properties.getCorePoolSize(),  
                properties.getMaxPoolSize(),  
                properties.getKeepAliveTime(),  
                TimeUnit.SECONDS,  
                new LinkedBlockingQueue<>(properties.getBlockQueueSize()),  
                Executors.defaultThreadFactory(),  
                handler);  
    }  
}
```

>[!info] com.ryan.config.ThreadPoolConfigProperties.java
```java
@Data  
@ConfigurationProperties(prefix = "thread.pool.executor.config", ignoreInvalidFields = true)  
public class ThreadPoolConfigProperties {  
    /** 核心线程数 */  
    private Integer corePoolSize = 20;  
    /** 最大线程数 */  
    private Integer maxPoolSize = 200;  
    /** 最大等待时间 */  
    private Long keepAliveTime = 10L;  
    /** 最大队列数 */  
    private Integer blockQueueSize = 5000;  
    /*  
     * AbortPolicy：丢弃任务并抛出 RejectedExecutionException 异常。  
     * DiscardPolicy：直接丢弃任务，但是不会抛出异常  
     * DiscardOldestPolicy：将最早进入队列的任务删除，之后再尝试加入队列的任务被拒绝  
     * CallerRunsPolicy：如果任务添加线程池失败，那么主线程自己执行该任务  
     */    
     private String policy = "AbortPolicy";  
}
```
