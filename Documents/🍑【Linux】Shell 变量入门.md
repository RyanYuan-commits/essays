## Shell 的变量
变量用于存储管理临时的数据，这些数据都是在运行内存中的；
它有三种类型：系统环境变量、自定义变量、特殊符号变量
### 环境变量
是系统提供的 **共享** 变量，是 Linux 系统加载 Shell 的配置文件中定义的变量共享给所有的 Shell 程序使用；
Shell 的全局配置文件有这些，Shell 环境加载这些变量给所有的用户所有的 Shell 的程序使用
- `/etc/profile`
- `/etc/profile.d/*.s`
- `/etc/bashrc`
Shell 个人配置文件有，Shell 环境加载个人配置文件中的变量给 **当前用户** 的 Shell 程序使用：
- `当前用户/bash_profile`
- `当前用户/.bashrc`
```sh
env
```
>查看当前系统对环境变量（用户级 + 系统级）
![[Linuxenv命令.png]]

```sh
set
```
>查看 Shell 变量（系统环境变量、自定义变量、函数）
![[Linuxset命令.png]]

| 变量名称     | 含义                                    |
| -------- | ------------------------------------- |
| PATH     | 与windows环境变量PATH功能一样，设置命令的搜索路径，以冒号为分割 |
| HOME     | 当前用户主目录：/root                         |
| SHELL    | 当前shell解析器类型：/bin/bash                |
| HISTFILE | 显示当前用户执行命令的历史列表文件：/root/.bash_history |
| PWD      | 显示当前所在路径：/root                        |
| OLDpwd   | 显示之前的路径                               |
| HOSTNAME | 显示当前主机名：iZ2ze3yq0kzs20azx7d2i9Z       |
| HOSTTYPE | 显示主机的架构，是i386、i686、还是x86、x64等：x86_64  |
| LANG     | 设置当前系统语言环境：zh_CN.UTF-8                |
常用的系统环境变量
```sh
cat /root/.bash_history
```
>查看当前用户历史执行命令
### 自定义变量
分为自定义局部变量、自定义常量和自定义全部变量

#### 自定义局部变量
定义在一个脚本文件中对变量，只能够在这个脚本中那种使用的变量

==变量定义==语法
```sh
[var_name]=value
```

- 变量名称可以用字母、数字和下划线组成，但是不能以数字开头，不能使用关键字作为变量名
- 等号两侧不能有空格
- 在 bash 环境中，变量的默认类型都是字符串类型，无法直接进行数值运算
- 变量的值如果有空格，必须使用双引号扩起来

==获取变量==的两种语法，第二种花括号的方式适合拼接字符串h
```sh
$变量名
${变量名}

echo my name is $name
	my name is ryan
echo my name is ${name}123
	my name is ryan123
```

==变量的删除==
```sh
unset [变量名]
```

#### 自定义常量
设置值以后不能修改对变量叫做常量，也叫只读变量
```sh
readonly [常量名]
```
![[linux自定义常量.png]]
#### 自定义全局变量
关于==父子 Shell 环境==：
	例如有两个 Shell 脚本文件，a.sh 和 b.sh，如果在 a.sh 中执行了 b.sh 文件，那么 a.sh 就是父 Shell 环境；
	b.sh 就是子 Shell 环境。
自定义全局变量可以在当前 Shell 环境和子 Shell 环境中都能使用。

```sh
export [变量名] [变量名]=value
```
> 可以导出已经创建好的变量，也可以在导出的时候创建变量；

demo2.sh
```sh
export NAME="ryan"
sh ./demo3.sh
```

demo3.sh
```java
echo $NAME
```

### 特殊变量
#### $n
`$n` ，n 为数字，用于接收 Shell 脚本执行的时候传入的参数；
`$0` 用于获取脚本名称；
`$1 ~ $9` 获取第一个输入参数到第九个输入参数；
第十个参数以后要使用 `${数字}` 来接收。

通过
```sh
sh 脚本文件 输入参数1 输入参数2
```
来执行文件的时候传入参数。

demo4.sh
```sh
echo name=$0
echo var1=$1
```
![[Linux特殊变量$n.png]]

#### $\#
输出所有输入参数的个数
```sh
echo name=$0
echo var1=$1
echo $#
```
![[Linux特殊变量.png]]
#### $\* 与 $@
```sh
$*
$@
```
> 都是获取所有的输入参数，用于输出所有参数
> 不使用双引号括起来，两个特殊变量效果相同，格式为 $1 $2 $3 ......
> 如果使用双引号括起来，`"$*"` 将获取的所有参数拼接为一个字符串，格式为："$1 $2 $3"
> 而 `"$@"` 获取的是一组参数列表对象，格式为 "$1" "$2" "$3"
```sh
for var in 列表变量
do # 循环开始
	命令 # 循环体
done # 循环结束
```

demo4.sh
```sh
echo name=$0
echo var1=$1
echo $#
echo '使用$*直接输出:'$*
echo '使用$@直接输出:'$@
for var in "$@"
do
 echo $var
done
for var in "$*"
do
 echo $var
done
```
执行 `sh ./demo4.sh hello world`：
![[Linux特殊变量$*与$@.png]]
#### $?
用于获取上一个 Shell 命令的退出状态码，或者函数状态值；
每个 Shell 都有一个返回值，用于说明命令执行是否成功，一般来说返回 0 表示成功，非 0 表示执行失败
![[Linux 特殊变量$?.png]]
#### \$$
用于获取当前 Shell 环境的进程 ID 号
```sh
ps -aux | grep bash
```
> 使用 ps 命令获取 bash 的进程 ID 号

也可以使用 `$$`
![[Linux 特殊变量$$.png]]

---

| 特殊变量    | 含义                                                                                          |
| ------- | ------------------------------------------------------------------------------------------- |
| $n      | 获取输入参数的 <br> $0，获取当前Shell脚本文件名字 <br> $1~$9，获取第一个输入参数到第九个输入参数 <br> ${10} 获取10和10以上的参数需要使用花括号 |
| $#      | 获取所有输入参数的个数                                                                                 |
| $* 与 $@ | 获取所有输入参数数据 <br>区别：如果不使用双引号，功能一样， 获取所有参数数据为一个字符串，<br>如果使用了双引号， $@ 获取的就是参数列表对象，每个参数都是一个独立字符串  |
| $?      | 获取上一个命令的退出状态码，一般 0 代表命令成功，非0 代表执行失败                                                         |
| $$      | 获取当前shell环境进程的ID号                                                                           |
